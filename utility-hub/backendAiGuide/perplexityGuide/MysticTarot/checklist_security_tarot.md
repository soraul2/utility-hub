3) checklist_security_tarot.md – Mystic Tarot Security Checklist v0.1
목적

Mystic Tarot 백엔드/클라이언트가 지켜야 할 보안·프라이버시 최소 기준을 체크리스트 형태로 정의.

1. 데이터 수집·보관
 수집 항목(이름·나이·성별·질문·리딩·시간 등)을 문서화.

 프로필 없이도 리딩 가능.

 히스토리/프로필 삭제 시 관련 데이터 실제 삭제.

 보관 정책 정의(무기한 or 최대 N년).

2. 전송·저장 보안
 HTTPS만 허용, HTTP 차단/리다이렉트.

 DB 비밀 값/LLM 키는 환경 변수·비밀 관리 도구 사용.

 로그에 민감 정보 저장 금지 또는 마스킹.

 LLM 요청/응답 전문은 저장 안 하거나 익명화.

3. 인증·권한·레이트 리밋
 관리자용 API는 Spring Security 인증·인가 적용.

 퍼블릭 리딩 API에 레이트 리밋.

 Actuator/Swagger 등 관리 엔드포인트 보호.

4. 입력 검증·API
 이름/나이/질문에 길이·형식 제한.

 XSS/스크립트 인젝션 방지 처리.
​

 에러 응답에 내부 스택트레이스 노출 금지.

5. AI 연동 가드레일
 시스템 메시지로 미래 보장 금지, 차별 표현 금지, 전문가 상담 권장 규칙 고정.

 응답 스키마 검증 + 폴백 메시지.

 LLM 요청에 불필요한 신상(이메일, 정확한 생일, 주소 등) 포함 금지.

6. 프라이버시·사용자 통제
 앱 내 프라이버시 안내/데이터 사용 설명 화면 제공.

 설정에서 프로필 수정/삭제, 히스토리 전체 삭제, 익명 모드 제공.

 마켓 Data Safety/Privacy 폼을 실제 구현과 일치시킴.

7. 테스트·운영
 보안/프라이버시 시나리오 테스트 케이스 작성 및 주기적 실행.

 분기별 보안·프롬프트 점검 및 변경 이력 기록.
