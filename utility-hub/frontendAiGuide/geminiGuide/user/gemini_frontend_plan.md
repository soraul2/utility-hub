# 프론트엔드 인증 구현 및 로그인 페이지 계획서

이 문서는 Gemini 팀이 Utility Hub 프론트엔드 인증 및 로그인 시스템을 구축하기 위한 계획과 산출물 명세를 정의합니다.

---

## 1. 프론트엔드 인증 구현 계획

### 1.1 아키텍처 설계
- **상태 관리**: `AuthContext`를 통해 전역적으로 로그인 상태를 유지합니다.
- **토큰 플로우**:
    1. 사용자가 소셜 로그인 버튼 클릭 → 백엔드 리다이렉트
    2. 소셜 인증 완료 후 프론트엔드 콜백으로 복귀 (`AuthCallbackPage`)
    3. 프론트엔드에서 백엔드 로그인 API 호출 후 토큰 수신
    4. 토큰을 저장하고 보호된 앱 공간으로 진입
- **자동 갱신**: Axios Interceptor를 사용하여 401 에러 발생 시 자동으로 토큰을 갱신합니다.

### 1.2 로그인 페이지 구성 방식
- **위치**: `/login` (LoginPage.tsx)
- **디자인 컨셉**:
    - **Premium Vibe**: Glassmorphism 디자인 적용, 부드러운 애니메이션.
    - **소셜 버튼**: 네이버(Green), 구글(White/Border) 브랜드 컬러를 살린 세련된 버튼 배치.
    - **UX**: "나만의 규칙과 기록을 안전하게 관리하세요"와 같은 환영 문구 포함.
    - **반응형**: 모바일과 데스크톱 모두에서 최적화된 레이아웃.
