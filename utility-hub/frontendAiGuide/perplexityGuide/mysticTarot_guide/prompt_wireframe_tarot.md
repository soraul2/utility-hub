3) prompt_wireframe_tarot.md – 질문·프로필 입력 & 백엔드 전달 구조 v0.1
0. Overview
목적: 사용자가 질문 + 현재 상황(선택) 을 자연스럽게 입력하고, 이를 백엔드가 LLM 프롬프트에 활용할 수 있게 일관된 JSON 구조로 전달.

원칙:

이름/나이/성별은 모두 선택 사항.

이 정보는 톤·예시 조정용으로만 쓰이며, 차별·고정관념 표현에 활용하지 않는다.
​

1. 질문 입력 UX 와이어
질문 입력 영역

Label: “어떤 점이 가장 궁금하신가요?”

Placeholder: “예: 올해 상반기 취업운이 궁금해요.”

타입: 멀티라인 TextArea.

주제 선택 (Topic)

Label: “어떤 주제에 가까운가요?”

옵션:

연애/관계 → LOVE

재물/금전 → MONEY

커리어/진로 → CAREER

건강/컨디션 → HEALTH

기타 → GENERAL

프로필 섹션

섹션 타이틀: “당신의 현재를 조금만 알려줄래요? (선택)”

이름

Label: “어떻게 불러드리면 좋을까요? (선택)”

Placeholder: “예: 민수, 별명도 좋아요”

나이

Label: “나이를 알려주시면, 지금 시기에 맞는 예시를 들 수 있어요. (선택)”

Input: 숫자.

성별

Label: “성별을 알려주시면 말투를 살짝 맞출게요. (선택)”

옵션 버튼: “여성 / 남성 / 말하고 싶지 않음”.

내부 값 매핑:

여성 → "FEMALE"

남성 → "MALE"

말하고 싶지 않음 → "UNKNOWN".
​

전송 버튼

라벨: “3장 스프레드 뽑기”.

클릭 시 validation:

질문 필수, 나머지는 모두 optional.

비활성 조건: 질문 공백.

2. 백엔드 전달 JSON 구조 (v0.1)
프론트에서 /readings/three-cards 호출 시 사용:

json
{
  "question": "올해 연애운이 어떨까요?",
  "userName": "홍길동",
  "userAge": 25,
  "userGender": "MALE",
  "topic": "LOVE"
}
question: 사용자의 자연어 질문 문자열.

topic: UI에서 선택한 주제 enum.

userName, userAge, userGender: 미입력 시 필드 자체를 보내지 않거나 null로 보낼 수 있도록 구현 (백엔드 허용 범위에 맞춰 결정).
​

3. LLM 프롬프트에 전달될 컨텍스트 (개념 설계용)
프론트는 LLM 프롬프트를 직접 만들지 않지만, 백엔드가 어떤 식으로 쓸지를 고려한 구조:

카드 정보:

카드명, 아르카나, 정/역 여부, 키워드, 의미(정/역).

사용자 정보 스냅샷:

이름: “민수님”처럼 부드럽게 호칭할 수 있도록 문자열 그대로.

나이: 20대, 30대 등으로 백엔드에서 구간 매핑 후 “진로를 고민할 시기”, “두 번째 커리어를 고민할 시기” 같은 예시에 활용.
​

성별: 말투·호칭에 아주 약하게만 반영, 고정관념 금지.

이걸 전제로 백엔드는 TarotAiService에서:

질문, topic, 카드 정보, 사용자 스냅샷을 합쳐 “Mystic” 페르소나용 프롬프트를 생성.
​

프론트는 단일 aiReading Markdown만 받아 사용자에게 보여준다.
​

4. 윤리/가드레일 관련 UI 메모
질문 폼 하단에 disclaimer 고정:

예: “이 리딩은 참고용 조언이며, 건강·재정·법률 등 중요한 결정은 반드시 전문가와 상의하세요.”
​

건강/재정/법률 주제 선택 시:

추가 안내 라벨을 작게 노출:

“중요한 의료·법률·재정 결정은 이 리딩을 근거로 내리지 마세요.”
